# Contributing to ctfscan

Thank you for your interest in contributing to ctfscan!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/ctfscan.git`
3. Create a virtual environment: `python -m venv venv`
4. Activate it: `source venv/bin/activate` (or `venv\Scripts\activate` on Windows)
5. Install in development mode: `pip install -e .`

## Making Changes

1. Create a new branch for your feature: `git checkout -b feature/my-feature`
2. Make your changes
3. Test your changes: `python -m pytest` (if tests exist)
4. Commit your changes with clear messages: `git commit -m "Add feature: description"`
5. Push to your fork and create a Pull Request

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings to functions
- Add comments for complex logic

## Reporting Bugs

Use the GitHub Issues tab to report bugs. Please include:
- Python version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior

## Feature Requests

Feel free to open an issue with the label "enhancement" to suggest new features.
